    モジュール名: 引数

# クラス図の書き方とは
 
`関連`：クラス間の関係性を記述します。  
勤怠管理システムの例だと、「社員」と「会社」クラス間には雇用関係がありますよね。  
その時は、クラス間を線で結び、線の上に関連名を記述します。  
 
`多重度`：「Aクラスから見た場合のBクラスが存在し得る数」です。  
「会社」クラスから見ると「社員」クラスは（従業員が1人以上存在する限り）複数存在するはずです。  
逆もしかりで、「社員」クラスから見て「会社」クラスは1つしか存在しません。  
この場合、クラス間をつなぐ線の端に、多重度を示す記号を表記します。  
 
`誘導可能性`：クラス同士のやりとりの方向性を表現します。  
「社員」クラスから「会社」クラスの操作ができることはありえません。  
逆に、「会社」クラスから「社員」クラスへのアクセスは考えられます。  
この場合、クラス間の関係を表す線は矢印とバツで表現します。  
 
クラスの型を継承する`汎化``特化``実現`とは：  
あるクラスAの型を継承してより具体化したクラスBを作成した場合、この2クラスの関係性は具体的なクラスBと抽象的なクラスAという意味で`汎化`関係と言えます。  
「営業社員」「事務社員」「技術社員」などのクラスを、より抽象的な「社員」クラスへ汎化することが可能です。  
また、逆に「社員」クラスからより具体的な「営業社員」「事務社員」「技術社員」などのクラスに具体化することを`特化`と言います。
 
「インターフェース」ともつながりますが、共通操作を定義したクラスと個別動作を定義したクラス間の関係を`実現`と言います。

クラスの`集約`と`コンポジット`とは  
クラス同士の関係が「全体と部分」であるときに`集約`として表現されます。  
「会社」クラスという全体があり、「社員」暮らすという部分があるイメージです。  

`コンポジット`は集約の一種で、「全体」クラスが「個別」クラスの生成や削除の権限があるときに使用されます。  
「会社」クラスが「社員」クラスを自由に生成・削除可能な場合は`コンポジット`であると言えます。
 
 クラス図の書き方とは。初心者にもわかりやすく解説  
 https://cacoo.com/ja/blog/how-to-write-class-diagram/

# コンピュータの処理速度
コンピュータの計算はCPUで行われます。  
ここで、命令を実行する単位を1サイクルといいます。
```
1.命令フェッチ(Fetch)：命令を主記憶から取り出し、CPUの命令レジスタに取り込む
2.命令の解読(Decode)：命令コードを解読する（デコード）
3.実行アドレス計算(Address Calculation)：命令対象となるオペランドの格納場所を計算して求める。
4.オペランド読み出し(Read)：主記憶の実行アドレスにアクセスし、オペランドを読み出す
5.命令の実行(Execution)：命令を実行し、必要に応じて結果データを主記憶に書き込む
```
https://itmanabi.com/cpu-basis/

1サイクルはおおよそ0.3ナノ秒かかります。

|処理|処理時間|倍率|
|---|---|---|
|CPU 1サイクル|0.3ナノ秒|base|
|floatの足し算|1.2ナノ秒|4倍|
|メモリアクセス|120ナノ秒|100倍|
|ストレージアクセス|100マイクロ秒|120,000倍|
|ネットワークアクセス|100ミリ秒|120,000,000倍|

1マイクロ = ナノの1000倍  
1ミリ = マイクロの1000倍  
100ミリ秒：人間のまばたきがおおよそ100ミリ秒くらいと言われています。 

イメージをしやすくするためにCPUの1サイクルのスケールをあげます。  
コンピュータの1サイクルを1秒としたときの比較  
|処理|処理時間|倍率|
|---|---|---|
|CPU 1サイクル|1秒|base|
|floatの足し算|4秒|4倍|
|メモリアクセス|およそ6分|100倍|
|ストレージアクセス|およそ4日|120,000倍|
|ネットワークアクセス|およそ10年|120,000,000倍|

# 法線
normal  
曲線状の接円と接線が定義される点で接線に垂直な直線を法線という。  
平面曲線では一意に定まるが三次元では主法線（接円の面内）、陪（従）法線（同に垂直）を区別する。  
曲面の接平面と直交する直線を、面の法線といい、接点を通り接平面に含まれる任意位の直線に対する垂線という。

# Pythonライブラリ

        - 分類

    - MediaPipeとは、Google社が提供するストリーミングメディア（動画等）を利用した学習パイプラインを構築するためのフレームワーク
        - 機械学習パイプラインとは、データ収集→データ処理→モデル（再）学習→再学習済モデルによる推論、といった一連の処理を自動化したシステム
    - 顔の形状推定、顔検出、ハンドトラッキング、物体検出等における高精度なモデルを提供。高精度。
    - メディア周りの機械学習を実装した→MediaPipe

    - 初学者が、PythonでAPI開発を行うには間違いなくFastAPI一択
